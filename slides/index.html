O que é um Framework?

O que é um Framework?

É uma arquitetura "padrão" que tem como objetivo fornecer ferramentas comuns a todo tipo de projeto a fim de proporcionar um ambiente de desenvolvimento mais produtivo.

Vantagens e Desvantagens em usar Framework

Vantagens e desvantagens

Vantagens:

  • Velocidade no desenvolvimento
  • Projeto padronizados
  • Mais segurança na aplicação
  • Facilita a manutenção
  • Fornece utilitários e bibliotecas

Desvantagens:

  • Performance e peso do projeto
  • Curva de aprendizado

Framework PHP de código aberto que utiliza o padrão de projeto MVC.

Principais características

  • Padrão de projetos MVC
  • Geração de CRUD (Create, Read, Update and Delete)
  • Utilização de templates

Definição das ferramentas

  • Linguagem PHP.

    É uma linguagem de script open source de uso geral.

  • Programação Orientada a Objetos.

    É um padrão de desenvolvimento para trabalhar com classes, métodos e atributos.

  • MVC (Model, View e Controller)

    Padrão de projetos que separa o sistema em camadas.

    • Model (Modelo)

    • View (Visão)

    • Controller (Controle)

  • Banco de dados Mysql.

    É um gerenciador de banco de dados relacional.

  • PDO (PHP Data Objects)

    Módulo PHP responsável por fazer a conexão e integração de dados entre o sistema e o banco de dados de forma mais segura.

  • Git

    Sistema de controle de versão.

Sistema (System)

Sistema (System)

Core do Framework. Local onde ficam todas as classes responsáveis pelo seu funcionamento.

helpers

São classes auxiliares do projeto.



Exemplo: Para uma classe helper de Email a mesma precisa se nomeada como EmailHelper.

Aplicação (App)

Aplicação (App)

Views

Para cada controller é criado uma pasta dentro da pasta Views.

Exemplo:

Para um controller noticias, é criada a pasta app/Views/Noticias.

Esta pasta guarda todas as views para esse controller.

As Views possuem extensão .phtml, como os arquivos de template.

Lib

Agrupa classes ou bibliotecas específicas do projeto. O acesso a essas classes fica facilitado com a utilização de uma constantes do framework denominada LIB.

Exemplo:
include LIB . classe.php

Webroot

É a pasta que passa a ser a raiz do projeto pelo servidor.
